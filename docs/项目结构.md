# 项目结构说明

这份说明用于约定项目目录的职责边界，避免入口、实验脚本、临时文件和资源文件混在根目录。

## 根目录

- `main.py`：应用启动入口，日常运行只需要执行这个文件。
- `install_runtime.bat`：Windows 首次安装脚本，创建 `.venv`、安装依赖并下载默认 RVM 模型。
- `start_upper_computer.bat`：Windows 一键启动上位机脚本。
- `requirements.txt`：上位机运行依赖列表。
- `README.md`：项目使用说明和快速索引。

根目录只保留项目级文件，不再放调试脚本、实验脚本、数据集或临时产物。

## 核心应用

- `algorithms/`：RVM 默认运行链路、统一工厂封装，以及保留的历史算法实现。
- `config/`：统一配置。项目根目录、资源目录、固定算法、模型路径、数据集路径和实验输出路径都集中在 `config/constants.py`。
- `gui/`：桌面界面代码。
- `utils/`：通用文件和图像工具。

## 资源与模型

- `assets/backgrounds/`：应用可选择的背景图片。
- `assets/screenshots/`：运行时截图输出，已加入 `.gitignore`。
- `assets/recordings/`：运行时录屏输出，已加入 `.gitignore`。
- `models/`：本地模型权重目录。当前默认上位机需要 `rvm_mobilenetv3_fp32.onnx`，权重文件通过 `.gitignore` 排除，可由 `install_runtime.bat` 首次安装时自动下载。

## 脚本与测试

- `scripts/debug/`：问题定位脚本，例如检查算法内部接口、检查 PPM-100 数据集、查看 MediaPipe 返回结构。
- `scripts/evaluation/`：实验和评估脚本，例如直播场景模拟、摄像头性能实验、PPM-100 指标评估和结果汇总。
- `tests/manual/`：需要手动运行的冒烟验证脚本。
- `tests/fixtures/`：测试和评估用的小型固定资源，例如绿色背景图。

这些脚本都通过 `Path(__file__).resolve().parents[2]` 定位项目根目录，因此从项目根目录执行即可正常导入核心模块。

## 数据与实验输出

- `data/PPM-100/`：本地 PPM-100 数据集默认位置，目录结构为 `image/` 和 `matte/`。
- `experiments/results/`：实验结果默认输出位置。

本地数据集和实验结果通常体积较大，已加入 `.gitignore`。如数据集不在默认位置，可使用环境变量覆盖：

```powershell
$env:PPM100_DIR="D:\datasets\PPM-100"
```

也可以分别设置：

```powershell
$env:PPM100_IMAGE_DIR="D:\datasets\PPM-100\image"
$env:PPM100_MATTE_DIR="D:\datasets\PPM-100\matte"
```
