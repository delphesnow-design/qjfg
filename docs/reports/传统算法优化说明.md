# 传统算法优化说明

> 说明：本报告用于记录 5 个传统 CV 算法的优化过程。后续补入真实人像直播压力测试后，默认上位机算法已切换为 RVM；传统算法保留为无模型环境、固定机位和历史对比基线。

## 优化依据

- MOG2/KNN：OpenCV 文档说明 `learningRate` 控制背景模型学习速度，`detectShadows` 会把阴影标成独立值；本项目做虚拟背景时，shadow 标签经常落在人物区域，因此优化为“预热期快速学习背景，运行期慢速学习，并把非零 mask 作为候选前景”。
- GrabCut：OpenCV 和原始 GrabCut 论文都强调先用矩形/trimap 初始化，再迭代图割。这里改为首次矩形初始化，后续用上一帧 alpha 构造 trimap，并降低处理分辨率做追踪。
- LOBSTER/SuBSENSE：LBSP 论文强调小邻域二进制相似模式适合低成本背景减除；原实现的汉明距离用 `np.unpackbits`，每帧开销较高，优化为 8-bit bitcount 查表。

参考来源：

- OpenCV MOG2 文档：https://docs.opencv.org/3.4/javadoc/org/opencv/video/BackgroundSubtractorMOG2.html
- OpenCV KNN 文档：https://docs.opencv.org/4.x/db/d88/classcv_1_1BackgroundSubtractorKNN.html
- OpenCV GrabCut 教程：https://docs.opencv.org/4.x/dd/dfc/tutorial_js_grabcut.html
- GrabCut 论文：Rother, Kolmogorov, Blake, 2004, `"GrabCut": Interactive Foreground Extraction using Iterated Graph Cuts`，https://www.microsoft.com/en-us/research/wp-content/uploads/2004/08/siggraph04-grabcut.pdf
- LBSP 论文：Bilodeau et al., 2013, `Change Detection in Feature Space Using Local Binary Similarity Patterns`，https://doi.org/10.1109/CRV.2013.29

## 代码改动

| 方法 | 优化点 |
| --- | --- |
| MOG2 | 调整 `history`、`varThreshold`、shadow 阈值、背景比例；加入预热/运行期 learning-rate；shadow 标签不再直接丢弃。 |
| KNN | 调整 `history`、`dist2Threshold`、样本数和 KNN 匹配数；同样采用预热/运行期 learning-rate 和 shadow 候选前景。 |
| GrabCut | 首帧仍用矩形初始化；后续帧用上一帧 alpha 生成 trimap，只跑少量迭代；降低 `process_max_side` 提升速度。 |
| LOBSTER | LBSP 汉明距离改为 bitcount 查表；边界采样由 wrap 改为 reflect，避免边缘误匹配。 |
| SuBSENSE | 复用同一 LBSP/汉明距离优化，保持原有自适应阈值逻辑。 |

## 本地 benchmark 对比

数据来自 `python scripts/evaluation/method_benchmark_report.py` 的离线合成帧测试，优化前为本轮改动前的同脚本结果，优化后为 2026-05-08 15:49:28 的结果。

| 方法 | 优化前 ms/帧 | 优化后 ms/帧 | 优化前 IoU | 优化后 IoU | 主要变化 |
| --- | ---: | ---: | ---: | ---: | --- |
| MOG2 | 4.19 | 4.21 | 0.863 | 0.997 | 速度基本持平，前景误替换大幅降低。 |
| KNN | 5.19 | 5.00 | 0.913 | 0.997 | 速度略升，mask 质量明显提升。 |
| GrabCut | 275.11 | 11.62 | 0.850 | 0.908 | 平均耗时大幅下降，IoU 同时提升。 |
| LOBSTER | 110.33 | 55.36 | 0.979 | 0.979 | 质量保持，速度约提升一倍。 |
| SuBSENSE | 113.12 | 58.17 | 0.961 | 0.961 | 质量保持，速度明显提升。 |

## 注意事项

这组优化以固定摄像头、先观察背景、再出现人物的实时背景替换场景为目标。若用于强动态背景、频繁光照突变或非中心人物构图，建议重新跑 `method_benchmark_report.py` 并按实际视频样本调参。
